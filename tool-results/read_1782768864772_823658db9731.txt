   278→## Round 4 — WebSocket Real-Time Price Streaming
   279→
   280→**Task ID:** 2
   281→**Agent:** fullstack-developer
   282→**Date:** 2025-01-xx
   283→
   284→### What Was Built
   285→
   286→A **WebSocket mini-service** using Socket.io that streams real-time crypto price updates to the frontend, replacing polling-only market data with live-streamed data.
   287→
   288→#### 1. Mini-Service (`mini-services/price-stream/`)
   289→- **`package.json`**: Independent bun project with `socket.io` dependency, port 3003, `bun --hot` dev script
   290→- **`index.ts`**: Socket.io server on port 3003 that:
   291→  - Fetches BTC/ETH/SOL prices from CoinGecko every 15 seconds
   292→  - Fetches Fear & Greed Index from Alternative.me
   293→  - Emits `price-update` events with `{ prices, updatedAt }` payload
   294→  - Emits `fear-greed-update` events with sentiment data
   295→  - Sends cached data immediately on client connect (zero-latency first paint)
   296→  - Handles graceful shutdown (SIGTERM/SIGINT)
   297→  - Logs connections, disconnections, and price fetches
   298→  - CORS enabled for all origins
   299→  - Uses `path: "/"` for Caddy gateway compatibility
   300→
   301→#### 2. React Hook (`src/hooks/use-price-stream.ts`)
   302→- Connects via `io("/?XTransformPort=3003")` (relative path + query param, NOT direct URL)
   303→- Listens for `price-update` and `fear-greed-update` events
   304→- Returns `{ prices, lastPriceUpdate, fearGreed, connectionStatus, reconnect }`
   305→- Exports TypeScript interfaces: `StreamPrice`, `StreamPriceUpdate`, `StreamFearGreedUpdate`, `ConnectionStatus`
   306→- Auto-reconnects via Socket.io built-in reconnection
   307→- Cleans up socket on unmount
   308→- Manual reconnect via `reconnect()` (increments state counter to re-trigger effect)
   309→
   310→#### 3. Dashboard Integration (`src/components/investor/dashboard.tsx`)
   311→- Imported and used `usePriceStream` hook
   312→- Added **LIVE indicator** in the fund overview banner:
   313→  - Green pulsing dot + "LIVE" when WebSocket connected with data
   314→  - Amber pulsing dot + "CONNECTING" during connection
   315→  - Red pulsing dot + "OFFLINE" when disconnected
   316→- Added **LIVE/DELAYED indicator** in Market Intelligence section header
   317→- Created merged data variables:
   318→  - `livePrices` = stream prices (if available) → fall back to polled API prices
   319→  - `liveSentiment` = stream fear/greed → fall back to polled API sentiment
   320→  - `isLive` = `connectionStatus === "connected" && streamPrices.length > 0`
   321→- Updated all market data consumers to use merged data:
   322→  - Market Ticker Bar (scrolling prices + F&G badge)
   323→  - Market Intelligence cards (BTC/ETH/SOL price cards)
   324→  - Market Sentiment gauge (Fear & Greed + BTC Dominance)
   325→
   326→### Files Created
   327→- `mini-services/price-stream/package.json` — new
   328→- `mini-services/price-stream/index.ts` — new
   329→- `src/hooks/use-price-stream.ts` — new
   330→
   331→### Files Modified
   332→- `src/components/investor/dashboard.tsx` — added usePriceStream hook, LIVE indicator, merged data
   333→- `package.json` — added `socket.io-client` dependency
   334→
   335→### Key Architecture Decisions
   336→- **Fallback pattern**: Stream data is preferred but the dashboard gracefully falls back to polled REST API data when WebSocket is disconnected. This ensures the UI always shows data.
   337→- **State-based reconnect**: Uses a `reconnectId` state counter instead of a ref to trigger effect re-runs, satisfying React linting rules.
   338→- **Immediate cache on connect**: Server sends cached prices on connect so clients don't wait 15 seconds for first data.

   994→- **Architecture decisions:**
   995→  - **Soft-link for documentId:** Per the task spec, `documentId` is a plain `String?` field on `StatementRequest` rather than a Prisma FK relation. Trade-off: slightly more query overhead but cleaner schema that doesn't require cascading deletes when a Document is removed.
   996→  - **Shared PDF generation helper:** Extracted the 150-LOC PDF generation logic from the existing `/api/admin/statements/generate` route into `src/lib/statement.ts` so both the direct admin generation flow and the new fulfill-request flow use the exact same code path.    997→  - **Dual notification on fulfill:** When an admin fulfills a request, the investor gets BOTH `new_document` (so the bell badge increments + toast says "New document ready") AND `new_message` (so they get a separate toast saying "Statement Request Completed"). This double-notification is intentional — the document event refreshes the Document Vault, the message event explicitly references the request being fulfilled.
