   143→   143→## Round 4 — WebSocket Real-Time Price Streaming
   144→   144→
   145→   145→**Task ID:** 2
   146→   146→**Agent:** fullstack-developer
   147→   147→**Date:** 2025-01-xx
   148→   148→
   149→   149→### What Was Built
   150→   150→
   151→   151→A **WebSocket mini-service** using Socket.io that streams real-time crypto price updates to the frontend, replacing polling-only market data with live-streamed data.
   152→   152→
   153→   153→#### 1. Mini-Service (`mini-services/price-stream/`)
   154→   154→- **`package.json`**: Independent bun project with `socket.io` dependency, port 3003, `bun --hot` dev script
   155→   155→- **`index.ts`**: Socket.io server on port 3003 that:
   156→   156→  - Fetches BTC/ETH/SOL prices from CoinGecko every 15 seconds
   157→   157→  - Fetches Fear & Greed Index from Alternative.me
   158→   158→  - Emits `price-update` events with `{ prices, updatedAt }` payload
   159→   159→  - Emits `fear-greed-update` events with sentiment data
   160→   160→  - Sends cached data immediately on client connect (zero-latency first paint)
   161→   161→  - Handles graceful shutdown (SIGTERM/SIGINT)
   162→   162→  - Logs connections, disconnections, and price fetches
   163→   163→  - CORS enabled for all origins
   164→   164→  - Uses `path: "/"` for Caddy gateway compatibility
   165→   165→
   166→   166→#### 2. React Hook (`src/hooks/use-price-stream.ts`)
   167→   167→- Connects via `io("/?XTransformPort=3003")` (relative path + query param, NOT direct URL)
   168→   168→- Listens for `price-update` and `fear-greed-update` events
   169→   169→- Returns `{ prices, lastPriceUpdate, fearGreed, connectionStatus, reconnect }`
   170→   170→- Exports TypeScript interfaces: `StreamPrice`, `StreamPriceUpdate`, `StreamFearGreedUpdate`, `ConnectionStatus`
   171→   171→- Auto-reconnects via Socket.io built-in reconnection
   172→   172→- Cleans up socket on unmount
   173→   173→- Manual reconnect via `reconnect()` (increments state counter to re-trigger effect)
   174→   174→
   175→   175→#### 3. Dashboard Integration (`src/components/investor/dashboard.tsx`)
   176→   176→- Imported and used `usePriceStream` hook
   177→   177→- Added **LIVE indicator** in the fund overview banner:
   178→   178→  - Green pulsing dot + "LIVE" when WebSocket connected with data
   179→   179→  - Amber pulsing dot + "CONNECTING" during connection
   180→   180→  - Red pulsing dot + "OFFLINE" when disconnected
   181→   181→- Added **LIVE/DELAYED indicator** in Market Intelligence section header
   182→   182→- Created merged data variables:
   183→   183→  - `livePrices` = stream prices (if available) → fall back to polled API prices
   184→   184→  - `liveSentiment` = stream fear/greed → fall back to polled API sentiment
   185→   185→  - `isLive` = `connectionStatus === "connected" && streamPrices.length > 0`
   186→   186→- Updated all market data consumers to use merged data:
   187→   187→  - Market Ticker Bar (scrolling prices + F&G badge)
   188→   188→  - Market Intelligence cards (BTC/ETH/SOL price cards)
   189→   189→  - Market Sentiment gauge (Fear & Greed + BTC Dominance)
   190→   190→
   191→   191→### Files Created
   192→   192→- `mini-services/price-stream/package.json` — new
   193→   193→- `mini-services/price-stream/index.ts` — new
   194→   194→- `src/hooks/use-price-stream.ts` — new
   195→   195→
   196→   196→### Files Modified
   197→   197→- `src/components/investor/dashboard.tsx` — added usePriceStream hook, LIVE indicator, merged data
   198→   198→- `package.json` — added `socket.io-client` dependency
   199→   199→
   200→   200→### Key Architecture Decisions
   201→   201→- **Fallback pattern**: Stream data is preferred but the dashboard gracefully falls back to polled REST API data when WebSocket is disconnected. This ensures the UI always shows data.
   202→   202→- **State-based reconnect**: Uses a `reconnectId` state counter instead of a ref to trigger effect re-runs, satisfying React linting rules.
   203→   203→- **Immediate cache on connect**: Server sends cached prices on connect so clients don't wait 15 seconds for first data.
   204→   204→
   205→   205→### Service Status
   206→   206→- Price-stream service running on port 3003
   207→   207→- Accessible via Caddy gateway at `/?XTransformPort=3003`

   290→   290→### Key Architecture Decisions

   529→   529→### Key Architecture Decisions
   530→   530→- **Broadcast efficiency**: Broadcast messages are stored as a single row with `isBroadcast=true` and `recipientId=null`, rather than duplicating for each investor. The investor API queries both broadcast messages and direct messages in a single `OR` query.
   531→   531→- **Auto-read on fetch**: The investor messages API automatically marks individual unread messages as read when fetched, avoiding a separate "mark as read" API call. Broadcast read status is tracked separately.
   532→   532→- **Health aggregation**: The overall health status is computed from 5 checks (DB healthy, DB latency, API latency, price feed, cache), giving "healthy" if all pass, "degraded" if ≥3 pass, "critical" otherwise.
   533→   533→- **Activity timeline**: Merges NAV updates, price feed events, and audit logs into a unified timeline for the admin health dashboard, giving a complete operational picture.
   534→   534→- **2FA adoption tracking**: The health endpoint calculates 2FA adoption rate from the User table, providing admins with visibility into security posture.
