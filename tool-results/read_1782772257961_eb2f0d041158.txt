   433→#### 1. Mini-Service (`mini-services/price-stream/`)
   434→- **`package.json`**: Independent bun project with `socket.io` dependency, port 3003, `bun --hot` dev script
   435→- **`index.ts`**: Socket.io server on port 3003 that:
   436→  - Fetches BTC/ETH/SOL prices from CoinGecko every 15 seconds
   437→  - Fetches Fear & Greed Index from Alternative.me
   438→  - Emits `price-update` events with `{ prices, updatedAt }` payload

   441→  - Handles graceful shutdown (SIGTERM/SIGINT)

   445→
   446→#### 2. React Hook (`src/hooks/use-price-stream.ts`)
   447→- Connects via `io("/?XTransformPort=3003")` (relative path + query param, NOT direct URL)
   448→- Listens for `price-update` and `fear-greed-update` events
   449→- Returns `{ prices, lastPriceUpdate, fearGreed, connectionStatus, reconnect }`
   450→- Exports TypeScript interfaces: `StreamPrice`, `StreamPriceUpdate`, `StreamFearGreedUpdate`, `ConnectionStatus`
   451→- Auto-reconnects via Socket.io built-in reconnection
   452→- Cleans up socket on unmount
   453→- Manual reconnect via `reconnect()` (increments state counter to re-trigger effect)
   454→
   455→#### 3. Dashboard Integration (`src/components/investor/dashboard.tsx`)
   456→- Imported and used `usePriceStream` hook
   457→- Added **LIVE indicator** in the fund overview banner:

   459→  - Amber pulsing dot + "CONNECTING" during connection
   460→  - Red pulsing dot + "OFFLINE" when disconnected
   461→- Added **LIVE/DELAYED indicator** in Market Intelligence section header
   462→- Created merged data variables:
   463→  - `livePrices` = stream prices (if available) → fall back to polled API prices
   464→  - `liveSentiment` = stream fear/greed → fall back to polled API sentiment
   465→  - `isLive` = `connectionStatus === "connected" && streamPrices.length > 0`
   466→- Updated all market data consumers to use merged data:
   467→  - Market Ticker Bar (scrolling prices + F&G badge)

   809→### Key Architecture Decisions
   810→- **Broadcast efficiency**: Broadcast messages are stored as a single row with `isBroadcast=true` and `recipientId=null`, rather than duplicating for each investor. The investor API queries both broadcast messages and direct messages in a single `OR` query.
   811→- **Auto-read on fetch**: The investor messages API automatically marks individual unread messages as read when fetched, avoiding a separate "mark as read" API call. Broadcast read status is tracked separately.
   812→- **Health aggregation**: The overall health status is computed from 5 checks (DB healthy, DB latency, API latency, price feed, cache), giving "healthy" if all pass, "degraded" if ≥3 pass, "critical" otherwise.
   813→- **Activity timeline**: Merges NAV updates, price feed events, and audit logs into a unified timeline for the admin health dashboard, giving a complete operational picture.
   814→- **2FA adoption tracking**: The health endpoint calculates 2FA adoption rate from the User table, providing admins with visibility into security posture.
